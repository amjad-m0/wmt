## 1. تنصيب البرامج المطلوبة

### أ) تنصيب Python

1. روح على: https://www.python.org/downloads/
2. حمّل **Python 3.11** أو **3.12** (مش أحدث من كده حالياً)
3. أثناء التنصيب على Windows: **لازم تعلّم على خانة "Add Python to PATH"** قبل ما تضغط Install
4. بعد التنصيب، افتح Terminal / Command Prompt واكتب:
   ```
   python --version
   ```
   المفروض يظهر لك: `Python 3.11.x` أو `3.12.x`

### ب) تنصيب PostgreSQL

1. روح على: https://www.postgresql.org/download/
2. اختار نظام التشغيل بتاعك (Windows / Mac / Linux)
3. حمّل ونصّب PostgreSQL (اختار **آخر نسخة مستقرة**، مثلاً 16)
4. أثناء التنصيب:
   - هيطلب منك **كلمة مرور لليوزر postgres** — اكتب كلمة مرور واحفظها (مثلاً: `postgres123`)
   - اسيب الـ **port** على القيمة الافتراضية: `5432`
   - لو ظهرت لك خانة "Stack Builder" في الآخر، اقفلها (مش محتاجينها)

### ج) تنصيب VS Code

1. روح على: https://code.visualstudio.com/
2. حمّل ونصّب البرنامج حسب نظام التشغيل بتاعك

### د) تنصيب الإضافات (Extensions) في VS Code

افتح VS Code، اضغط على أيقونة الـ Extensions (المربعات الأربعة على الشمال)، وابحث عن ونصّب:

| الإضافة | الاستخدام |
|---|---|
| **Python** (by Microsoft) | تشغيل وتتبع كود Python |
| **Pylance** (by Microsoft) | إكمال تلقائي وفحص أخطاء Python |
| **Jinja** (by wholroyd) | تلوين كود HTML/Jinja بتاع الصفحات |
| **SQLTools** (by Matheus Teixeira) — اختياري | لمعاينة قاعدة البيانات من جوه VS Code |
| **SQLTools PostgreSQL Driver** — اختياري | يشتغل مع SQLTools |

## 2. تجهيز مجلد المشروع

1. فك ضغط الملف اللي استلمته (WMS.zip) في أي مكان تحبه، مثلاً:
   - Windows: `C:\Projects\WMS`
   - Mac/Linux: `~/Projects/WMS`

2. افتح VS Code، ثم من القائمة: **File → Open Folder** واختار المجلد اللي فككت فيه الملف

3. افتح Terminal جوه VS Code من القائمة: **Terminal → New Terminal**
   (هتلاقي نفسك بالفعل جوه مجلد المشروع)

## 3. تجهيز قاعدة البيانات

في الـ Terminal بتاع VS Code (لازم يكون مسار الـ Terminal فيه أدوات PostgreSQL، غالباً موجودة تلقائياً بعد التنصيب):

### على Windows:

1. افتح **SQL Shell (psql)** من قائمة Start (بتتنصب مع PostgreSQL)
2. هيسألك عن Server, Database, Port, Username — اضغط Enter على كل واحدة (تسيبها افتراضية) لحد ما يطلب الباسورد
3. اكتب كلمة المرور اللي حطيتها وقت التنصيب
4. دلوقتي أنت جوه psql، اكتب الأوامر دي واحد واحد (اضغط Enter بعد كل سطر):

```sql
CREATE USER wms_user WITH PASSWORD 'wms_password';
CREATE DATABASE wms_db OWNER wms_user;
GRANT ALL PRIVILEGES ON DATABASE wms_db TO wms_user;
```

5. تأكد إن كل أمر طلع `CREATE ROLE` / `CREATE DATABASE` / `GRANT` بدون أخطاء
6. اخرج بكتابة: `\q`

### على Mac/Linux:

```bash
psql -U postgres
```
(هيطلب كلمة المرور بتاعة postgres)

بعدين نفس الأوامر اللي فوق:
```sql
CREATE USER wms_user WITH PASSWORD 'wms_password';
CREATE DATABASE wms_db OWNER wms_user;
GRANT ALL PRIVILEGES ON DATABASE wms_db TO wms_user;
\q
```

> **ملاحظة:** لو عاوز تغيّر اسم المستخدم أو كلمة المرور، لازم تحدّث نفس القيم في ملف `.env` لاحقاً (الخطوة 5).

## 4. تجهيز بيئة Python

في Terminal بتاع VS Code، وأنت جوه مجلد المشروع، اكتب الأوامر دي **بالترتيب**:

### إنشاء بيئة افتراضية (Virtual Environment)

```bash
python -m venv venv
```

### تفعيل البيئة الافتراضية

**على Windows (Command Prompt):**
```
venv\Scripts\activate
```

**على Windows (PowerShell):**
```
venv\Scripts\Activate.ps1
```
> لو ظهر خطأ صلاحيات في PowerShell، اكتب الأمر ده مرة واحدة بس ثم أعد المحاولة:
> ```
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

**على Mac/Linux:**
```bash
source venv/bin/activate
```

> بعد التفعيل، هتلاقي `(venv)` ظاهرة في بداية سطر الـ Terminal — ده معناه إن البيئة شغالة صح.

### تنصيب المكتبات المطلوبة

```bash
pip install -r requirements.txt
```

هيبدأ يحمّل وينصّب كل المكتبات (FastAPI, SQLAlchemy, إلخ) — ممكن ياخد دقيقة أو دقيقتين.

> **ملاحظة مهمة:** لو ظهر لك خطأ بخصوص `bcrypt`، نفّذ الأمر ده كمان:
> ```bash
> pip install bcrypt==4.0.1
> ```

## 5. ضبط ملف الإعدادات (.env)

جوه مجلد المشروع هتلاقي ملف اسمه `.env` — افتحه في VS Code وشوف القيم دي:

```
DATABASE_URL=postgresql://wms_user:wms_password@localhost:5432/wms_db
SECRET_KEY=change-this-to-a-long-random-string-in-production
```

- لو غيّرت اسم المستخدم أو كلمة السر بتاعة الـ database في الخطوة 3، حدّث `DATABASE_URL` بنفس القيم.
- غيّر `SECRET_KEY` لأي نص عشوائي طويل (مهم للأمان، خصوصاً قبل ما ترفع النظام على سيرفر حقيقي).

باقي القيم (Cloudflare R2) سيبها زي ما هي حالياً — النظام هيحفظ الصور محلياً على جهازك تلقائياً لحد ما تجهز حساب R2 (شرحناه في الوثيقة المرجعية).

## 6. إنشاء أول حساب مدير

في نفس الـ Terminal (والبيئة الافتراضية `(venv)` لسه شغالة)، اكتب:

```bash
python init_admin.py
```

هيسألك 3 أسئلة:
```
اسم المستخدم للمدير (username): admin
الاسم الكامل: [اسمك]
كلمة المرور: [اختار كلمة مرور قوية]
```

بعد ما تدخلهم هتلاقي رسالة:
```
✅ تم إنشاء حساب المدير: admin
```

> احفظ اسم المستخدم وكلمة المرور دول — هتستخدمهم لتسجيل الدخول.

## 7. تشغيل النظام

في نفس الـ Terminal:

```bash
uvicorn app.main:app --reload
```

هتشوف رسالة زي دي:
```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

دلوقتي افتح المتصفح واكتب:
```
http://localhost:8000
```

هيوديك على صفحة تسجيل الدخول — سجّل دخول بالـ username والباسورد اللي عملتهم في الخطوة 6.

> **لإيقاف السيرفر:** ارجع للـ Terminal واضغط `Ctrl + C`

> **لتشغيله تاني في أي وقت:** افتح Terminal في مجلد المشروع، فعّل الـ venv (الخطوة 4)، واكتب `uvicorn app.main:app --reload` تاني.

## 8. استخدام النظام من VS Code

كل مرة تفتح المشروع من جديد:

1. افتح مجلد المشروع في VS Code
2. افتح Terminal جديد (`Terminal → New Terminal`)
3. فعّل الـ virtual environment:
   - Windows: `venv\Scripts\activate`
   - Mac/Linux: `source venv/bin/activate`
4. شغّل السيرفر: `uvicorn app.main:app --reload`
5. افتح `http://localhost:8000` في المتصفح

### هيكل المشروع (عشان تعرف تتنقل فيه)

```
WMS/
├── app/
│   ├── main.py              ← نقطة بداية التطبيق
│   ├── config.py             ← الإعدادات العامة
│   ├── database.py           ← الاتصال بقاعدة البيانات
│   ├── auth.py                ← تسجيل الدخول والصلاحيات
│   ├── models/
│   │   └── models.py         ← كل جداول قاعدة البيانات
│   ├── routers/               ← كل صفحة/API في ملف منفصل
│   │   ├── auth.py
│   │   ├── materials.py       ← المواد والمخزون
│   │   ├── orders.py           ← أوامر الشغل والصيانة
│   │   ├── issuances.py        ← الإصدار والصرف
│   │   ├── borrowings.py       ← الاستلاف
│   │   ├── transfers.py        ← النقل الداخلي
│   │   ├── returns.py          ← المرتجعات
│   │   ├── search.py           ← البحث والـ Audit Log
│   │   └── admin.py            ← إدارة المستخدمين
│   ├── services/
│   │   ├── storage.py          ← رفع الصور (محلي أو R2)
│   │   ├── pdf_service.py       ← توليد وصولات PDF
│   │   └── audit_service.py     ← تسجيل كل التغييرات
│   ├── templates/               ← كل صفحات HTML (بالعربي RTL)
│   └── static/                  ← CSS و JS
├── requirements.txt              ← قائمة المكتبات المطلوبة
├── .env                           ← الإعدادات السرية (لا ترفعه على GitHub)
├── init_admin.py                  ← سكريبت عمل أول مدير
└── SETUP.md                        ← هذا الملف
```

### إزاي تعدّل حاجة

- عاوز تعدّل شكل صفحة؟ → روح على `app/templates/` ولاقي الصفحة المطلوبة
- عاوز تضيف حقل جديد لجدول؟ → عدّل `app/models/models.py` ثم شغّل الأوامر دي لتحديث قاعدة البيانات:
  ```bash
  python -c "from app.database import engine, Base; import app.models.models; Base.metadata.create_all(bind=engine)"
  ```
- عاوز تعدّل منطق عملية معينة (مثلاً الاستلاف)؟ → روح على `app/routers/borrowings.py`

## 9. حل المشاكل الشائعة

| المشكلة | الحل |
|---|---|
| `ModuleNotFoundError` | تأكد إن الـ `(venv)` شغالة (خطوة 4)، وأعد `pip install -r requirements.txt` |
| `password authentication failed` | راجع القيم في `.env` وتأكد إنها مطابقة لليوزر والباسورد اللي عملتهم في PostgreSQL |
| `Connection refused` على الـ database | تأكد إن خدمة PostgreSQL شغالة (ابحث عن "Services" على Windows وتأكد إن postgresql شغال) |
| السيرفر بيقفل لوحده | تأكد إنك مش قافل الـ Terminal بتاعه |
| صورة مش بترفع | تأكد إن مجلد `uploads` موجود جوه مجلد المشروع (بيتعمل تلقائياً أول ما ترفع صورة) |
| `bcrypt` error | نفّذ `pip install bcrypt==4.0.1` |
| الصفحة فاضية أو `Internal Server Error` | ارجع للـ Terminal وشوف رسالة الخطأ بالتفصيل — هتوضحلك السطر والملف اللي فيه المشكلة |

## 10. الخطوات القادمة

النظام دلوقتي شغال بالكامل على جهازك (localhost)، ومحدّث بآخر متطلبات العميل الموثقة (v6):

**آخر إضافات (v6):**
- **المستودع على مستوى الأمر**: أي أمر شغل/صيانة بقى فيه اختيار "المستودع المصدر" (الدمام 314 / بن قريعة 310) — قبل كده كان مربوط بالمادة الفردية بس.
- **الإضافة الجماعية للمواد**: صفحة جديدة "إضافة جماعية" لما يكون عندك أكتر من وحدة من نفس النوع (زي 3 RMU) — كل وحدة برقم تسلسلي وأمر عمل مختلف، في عملية واحدة بدل تكرار الفورم 3 مرات.
- **رقم السيريال وكود الصنف ظاهرين في صفحة الأمر**: جدول "المواد المسجلة على الأمر" بقى يوضح كود الصنف والرقم التسلسلي لكل مادة، مش بس الاسم والكمية.
- **إصلاح**: حقل "المشرف" في فورمات كانت لسه بتظهر كقائمة اختيار يدوية (أمر جديد + إضافة Gate Pass) — اتأكد إنها بقت تلقائية بالكامل حسب تسجيل الدخول.

**آخر إصلاحات (v5):** ربط حقيقي للمادة بالأمر (FK مش نص)، إصلاح خصم المخزون عند الإصدار، إصلاح قبول الاستلاف غلط، كود منفصل لأمر العمل عن الصيانة، رد الاستلاف بفورم كامل، زرار طباعة بس من غير خروج من الموقع، مشرف تلقائي في كل مكان، استرجاع العدة، إصلاح الأرقام العشرية.

**آخر إضافات (v4):** الطباعة العربية بخطوط مدمجة، الاستلاف/التسليف (داخلي/مقاول)، ربط مواد جديدة مباشرة مع الـ Gate Pass.

**آخر إضافات (v3):** الموردين الثابتين، فئة "معدة"، كود الصنف.

**آخر إضافات (v2):** شاشة الموردين، MDR، أنواع الصرف، شاشة فحص الأجهزة المعطلة، الكود التسلسلي الفريد.

الخطوة الجاية لما تكون جاهز:

1. **رفع النظام على سيرفر Hetzner** — عشان يبقى متاح لك ولفريقك من أي مكان
2. **ربط Cloudflare R2** — عشان الصور تتحفظ بشكل دائم وآمن بدل التخزين المحلي
3. **إعداد Backup تلقائي يومي**
4. **تفعيل HTTPS** عبر Let's Encrypt

كل التفاصيل دي موجودة في **الوثيقة المرجعية (WMS_Reference_Document_v2.docx)** تحت قسم "Infrastructure & Security" — لما تكون جاهز قولي وهنعمل جلسة Deploy كاملة خطوة بخطوة.
